# Naming Standards

## FORBIDDEN PROMOTIONAL WORDS
Never use these unprofessional, prejudicial terms in filenames:

- UNIFIED, REFACTORED, IMPROVED
- WORKING, FIXED, NEW
- COMPLETE, ENHANCED, ULTIMATE
- PERFECT, ADVANCED, COMPREHENSIVE
- BULLETPROOF, ULTRA, SUPER
- 4096D, MEGA, AWESOME

## PROPER NAMING CONVENTIONS

### Files:
- `status_report.md` (not WORKING_STATUS_REPORT.md)
- `api_guide.md` (not STYLE_ENHANCED_API_GUIDE.md)
- `echo_test.py` (not bulletproof_echo_test.py)
- `verification_report.md` (not bulletproof_verification_report.md)

### Variables:
- `search_engine` (not unified_search_engine)
- `model_manager` (not enhanced_model_manager)
- `conversation_handler` (not bulletproof_conversation_handler)

### Functions:
- `process_request()` (not process_request_perfectly())
- `handle_error()` (not ultimate_error_handler())
- `search_vectors()` (not comprehensive_vector_search())

## NAMING RULES
1. Use descriptive, functional names
2. No promotional/marketing language
3. No prejudicial adjectives
4. Focus on WHAT it does, not how "good" it is
5. Use snake_case for files and functions
6. Use PascalCase for classes only

## EXAMPLES

### BAD:
- `BULLETPROOF_ECHO_SYSTEM.py`
- `COMPREHENSIVE_ANALYSIS_REPORT.md`
- `ULTIMATE_VECTOR_SEARCH.py`
- `4096D_ULTRA_EMBEDDINGS.py`

### GOOD:
- `echo_system.py`
- `analysis_report.md`
- `vector_search.py`
- `embeddings.py`
